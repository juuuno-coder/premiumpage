# 전자카탈로그 프로젝트 OKR (Objectives and Key Results)

**작성일:** 2024-02-14
**최종 업데이트:** 2026-02-15
**기간:** Q1 2024 ~ (진행 중)
**버전:** 2.0.0

---

## 📊 프로젝트 개요

**현재 작업 중인 코드/기능:**
- HS-TECH, GENTOP, EMT, 항성산업사 등 다중 기업 전자카탈로그 플랫폼
- AI 에이전트 시스템을 활용한 자동화된 카탈로그 생성 및 관리
- 산업용 센서 및 측정 장비 전문 제품 카탈로그

**프로젝트의 최종 목적:**
- **누가:** 전 세계 산업용 센서 구매 담당자, 엔지니어, 기술 의사결정자
- **왜:** 복잡한 기술 사양을 빠르고 정확하게 비교하여 최적의 제품을 선택하기 위해
- **무엇:** 직관적이고 전문적인 디지털 카탈로그 경험

## 🎯 Objective 1: 모든 산업 엔지니어가 신뢰하고 찾는 글로벌 No.1 전자카탈로그 플랫폼을 구축한다

> "단순히 제품 정보를 나열하는 것이 아니라, 엔지니어가 30초 안에 원하는 제품의 핵심 사양을 이해하고 확신을 가지고 선택할 수 있게 한다."

### Key Results

**KR 1: 사용자 체류 시간 200% 증가**
- 현재: 평균 1분 30초
- 목표: 평균 4분 30초
- 측정 방법: Google Analytics 세션 지속 시간

**KR 2: 페이지 이탈률 50% 감소**
- 현재: 65% (첫 페이지에서 이탈)
- 목표: 32.5%
- 측정 방법: Google Analytics 이탈률 (Bounce Rate)

**KR 3: 기술 사양 정확도 100% 달성 및 이미지 매칭 완벽 구현**
- 현재: 약 85% (일부 이미지 누락, 사양 불일치)
- 목표: 100% (모든 제품 이미지 로딩, 공식 데이터시트 기준 사양)
- 측정 방법: 자동화된 검증 스크립트 + 수동 감사

**KR 4: 모바일 사용자 만족도 4.5/5.0 이상**
- 현재: 측정 중
- 목표: 평균 4.5/5.0
- 측정 방법: 사용자 피드백 설문 (5점 척도)

## 🎯 Objective 2: AI 에이전트 시스템으로 카탈로그 생성 시간을 10배 단축하고 품질을 2배 향상시킨다

> "수동으로 1주일 걸리던 카탈로그 작업을 AI 에이전트가 하루 만에 완성하되, 품질은 오히려 더 정확하고 일관성 있게."

**KR 1: 카탈로그 생성 시간 90% 단축**
- 현재: 신규 기업 카탈로그 생성 약 5-7일
- 목표: 12시간 이내 (1.5 근무일)
- 측정 방법: 에이전트 실행 로그 타임스탬프

**KR 2: 데이터 추출 정확도 98% 이상**
- 현재: 약 85% (수동 수정 필요)
- 목표: 98% (최소한의 검수만 필요)
- 측정 방법: 공식 데이터시트 대조 검증

**KR 3: 에이전트 병렬 실행으로 처리량 3배 증가**
- 현재: 순차 실행 (한 번에 1개 카탈로그)
- 목표: 동시 3개 카탈로그 처리 가능
- 측정 방법: 동시 작업 완료 건수

**KR 4: 코드 재사용률 80% 달성**
- 현재: 약 50% (기업별 중복 코드 존재)
- 목표: 80% (catalog-utils 공통화)
- 측정 방법: 코드 중복도 분석 도구

## 🎓 OKR 작성 원칙 재확인

### ✅ Good OKR
```
Objective: 모든 엔지니어가 신뢰하는 카탈로그 플랫폼
- 가슴 뛰고 영감을 줌
- 사용자 가치 중심
- 방향성이 명확

KR 1: 사용자 체류 시간 200% 증가
- 수치로 측정 가능
- 도전적 (70% 달성도 성공)
- Outcome 중심 (결과)
```

### ❌ Bad OKR
```
Objective: 웹사이트 기능 개선
- 영감 없음
- 너무 모호함
- 가치가 불명확

KR 1: 새로운 페이지 10개 추가
- Output 중심 (할 일)
- 비즈니스 가치 불명확
- 도전적이지 않음
```

### 핵심 구분: Output vs Outcome

| Output (할 일) | Outcome (결과) |
|--------------|--------------|
| 디자인 시안 3개 제작 | 구매 전환율 30% 증가 |
| 개발자 2명 채용 | 개발 속도 2배 향상 |
| 테스트 코드 작성 | 버그 발생률 50% 감소 |
| 이미지 100개 추가 | 사용자 만족도 4.5/5.0 |

**원칙: KR은 항상 Outcome으로 작성!**

## 💡 OKR을 성공적으로 운영하는 팁

### 1. 70% 달성을 목표로
- 100% 달성은 목표가 너무 쉬웠다는 의미
- 70-80% 달성이 이상적
- 도전적인 목표 설정의 증거

### 2. Output을 KR로 착각하지 말기
- "페이지 10개 추가" (X)
- "사용자 체류 시간 200% 증가" (O)

### 3. 주간 체크인 절대 생략 금지
- OKR은 분기말에 확인하는 게 아님
- 매주 진행 상황 점검
- 빠른 조정과 학습

### 4. 실패를 두려워하지 말기
- 70% 달성도 성공
- 실패한 OKR에서 더 많이 배움
- 다음 분기에 반영

### 5. 팀과 함께 만들기
- 리더가 일방적으로 할당 (X)
- 팀이 함께 토론하고 합의 (O)
- 주인의식 (Ownership) 형성
